# ELITE WORLD Project Rules

- حافظ دائماً على الهوية الحالية والشعار والألوان وRTL والتجاوب.
- لا تعِد بناء المشروع من الصفر.
- لا تحذف أو تستبدل المنتجات أو الصور أو البيانات أو Migrations الحالية إلا بطلب صريح.
- لا تشغّل Seed يمسح البيانات الموجودة.
- نفّذ فقط المهمة المطلوبة، ولا تضف مزايا جانبية.
- اقرأ وعدّل الملفات المرتبطة بالمهمة فقط.
- استخدم المكونات والبنية الموجودة، ولا تنشئ أنظمة مكررة.
- اجعل جميع وظائف الإدارة سهلة لشخص لا يملك خبرة برمجية.
- اجعل إضافة وتعديل وحذف المحتوى ممكنة من لوحة التحكم دون تعديل الكود.
- تجنب الإعدادات والتكاملات المعقدة، واجعل النشر وربط الدومين بسيطاً.
- لا تضع كلمات مرور أو مفاتيح حقيقية داخل الكود.
- عند غياب خدمة خارجية، استخدم إعداداً آمناً ورسالة واضحة بدلاً من تعطيل الموقع.
- لا تغيّر تصميم واجهة المتجر إلا إذا طلب ذلك صراحة.
- لا تشغّل TypeScript والاختبارات وBuild إلا مرة واحدة في نهاية المهمة.
- حدّث README وPROJECT_PROGRESS باختصار فقط عند وجود تغيير مهم.
- بعد كل مرحلة، أنشئ ZIP كاملاً للمشروع يشمل الصور والبيانات وPrisma والتوثيق.
- إذا كان الطلب غامضاً، اختر الحل الأكثر أماناً الذي لا يحذف البيانات، واذكر افتراضك في التقرير.
- اجعل التقرير النهائي مختصراً: ما تم، الفحوص، وأي خطوة يدوية مطلوبة.
