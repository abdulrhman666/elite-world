## قبل أي تعديل

- تأكد أن المستودع هو `abdulrhman666/elite-world` وأن الفرع المقصود هو `main` ما لم يطلب المالك غير ذلك.
- شغّل `git status --short --branch` وتوقف إذا وجدت تغييرات غير مفهومة؛ لا تمسح عملًا موجودًا.
- حدّد الملفات المرتبطة بالمهمة فقط. لا تعِد بناء النظام ولا تنشئ خدمة مكررة.
- افحص موضع الحقيقة الحالي قبل التعديل: Prisma للبيانات، `lib/catalog/service.ts` للكتالوج، `lib/payments/payment.service.ts` للدفع، وStorage Adapter للملفات.
- لا تقرأ قيم ملفات البيئة في التقارير، ولا تنسخها إلى كود أو Markdown أو Git.

## الخدمات المرتبطة

- GitHub: المستودع الحالي ومصدر نشر Vercel.
- Vercel: الاستضافة، النشر التلقائي من `main`، ومتجر Blob العام للصور.
- Neon PostgreSQL: قاعدة الإنتاج عبر Prisma.
- Resend: رموز تحقق البريد واستعادة كلمة المرور؛ التحقق البريدي يعمل حاليًا حسب اختبار المالك.
- OpenAI: اختياري فقط لمقترحات SEO عند وجود الإعداد، ولا يعتمد عليه الموقع.
- مزود الدفع: طبقة ديناميكية موجودة، لكن تفعيل مزود مالي حقيقي يحتاج تحققًا مستقلًا.

## تحقق بعد كل تعديل

- اختبر المسار الذي عُدّل يدويًا، مع حالة نجاح وحالة فشل.
- عند تغيير كود: شغّل مرة واحدة في النهاية `npm run typecheck` ثم `npm test` ثم `npm run build`.
- شغّل `npm run lint` و`npm run format:check` عندما تتغير ملفات TypeScript أو التنسيق.
- عند تغيير Prisma: راجع SQL الخاص بالـMigration يدويًا، ولا تشغّل Seed أو Migration مدمرة على الإنتاج.
- عند تغيير المصادقة: اختبر إنشاء الحساب، رمز البريد، الدخول، الخروج، الاستعادة، وصلاحيات `ADMIN` و`SUPER_ADMIN`.
- عند تغيير التجارة: اختبر المنتج ← السلة ← Checkout ← الطلب ← لوحة الإدارة ← رابط التتبع.
- عند تغيير الصور: اختبر الرفع، WebP، العرض، الصورة الرئيسية، والحذف الآمن في Vercel Blob.
- قبل التسليم شغّل `git diff --check`، افحص عدم وجود أسرار، وتأكد أن الـdiff محصور بالمهمة.

## ممنوع دون فهم وموافقة صريحة

- حذف أو استبدال المنتجات، الصور، بيانات Neon، ملفات CSV، أو Migrations.
- تشغيل Seed على الإنتاج أو إعادة إنشاء قاعدة البيانات.
- تغيير `AUTH_SECRET`؛ تغييره يبطل الجلسات وقد يمنع فك مفاتيح الدفع المشفرة.
- نقل مصدر البيانات من Prisma أو إضافة fallback تجريبي وقت التشغيل.
- تغيير الهوية أو RTL أو تصميم المتجر خارج نطاق الطلب.
- إضافة مزود دفع، OAuth، SMS، أو خدمة خارجية لم يطلبها المالك.
- تعديل متغيرات Vercel أو DNS أو Neon أو Resend تلقائيًا دون موافقة واضحة.
- الالتزام أو الرفع مع ملفات `.env` أو مفاتيح أو كلمات مرور أو بيانات عملاء.
