# ELITE WORLD — الكتالوج المرتبط بـCSV وPrisma

واجهة عربية RTL لمنصة معدات صناعية B2B، مبنية بـ Next.js وReact وTypeScript وTailwind CSS. تشمل الكتالوج وحسابات العملاء ولوحة الإدارة وعروض الأسعار والطلبات والتتبع والسلة وCheckout ونظام دفع متعدد المزودات دون تفعيل بوابة خارجية حقيقية.

## التشغيل

المتطلبات: Node.js 20.9 أو أحدث وnpm.

```bash
npm install
npm run dev
```

ثم افتح `http://localhost:3000`.

فحوص الجودة:

```bash
npm run typecheck
npm run lint
npm test
npm run format:check
npm run build
npm start
```

يستخدم أمر التطوير والبناء محرك webpack المضمّن في Next.js لأن Turbopack 16.2.9 يحتوي على خلل في التعامل مع الأحرف العربية داخل مسار المجلد الحالي. لا يغيّر ذلك واجهة الموقع أو إمكاناته.

## التقنية

- Next.js 16.2.9 مع App Router وReact 19.2.7.
- TypeScript 5.9 بوضع Strict.
- Tailwind CSS 4.3 ونظام Design Tokens مركزي.
- ESLint وPrettier وVitest.
- Prisma 7 وPostgreSQL لبيانات الكتالوج وإعدادات الموقع.
- خط IBM Plex Sans Arabic محلياً، وManrope للأرقام والموديلات.
- أيقونات Lucide خفيفة.
- Server Components افتراضياً، مع Client Components للتفاعلات الضرورية فقط.

## التخصيص

### بيانات التواصل

عدّل القيم Placeholder في `config/site.ts`: الاسم العربي، الهاتف، واتساب، البريد، المدينة، العنوان، ساعات العمل، وروابط التواصل.

انسخ `.env.example` إلى `.env` وضع رابط الموقع الحقيقي ورابط PostgreSQL:

```env
NEXT_PUBLIC_SITE_URL=https://example.com
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/elite_world?schema=public
DIRECT_URL=postgresql://postgres:postgres@localhost:5432/elite_world?schema=public
```

## قاعدة بيانات الكتالوج

نماذج Prisma تشمل الكتالوج والصور و`SiteSettings`، إضافة إلى `PageSeo` و`SlugRedirect` لإدارة SEO والتحويلات الدائمة. PostgreSQL هو مصدر بيانات التشغيل الوحيد:

```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/elite_world?schema=public
DIRECT_URL=postgresql://postgres:postgres@localhost:5432/elite_world?schema=public
```

عند استخدام Neon ضع رابط الـPooler في `DATABASE_URL` للتشغيل، والرابط المباشر في `DIRECT_URL` للمهاجرات. يستخدم `prisma.config.ts` الاتصال المباشر تلقائياً، بينما يستخدم التطبيق رابط الـPooler.

ثم نفّذ بالترتيب:

```bash
npm run prisma:generate
npm run prisma:migrate
npm run prisma:seed
```

- `catalog:import`: يتحقق من CSV ويولّد تقريري Markdown وJSON عند تحديث الكتالوج المستورد.
- `prisma:generate`: يولّد Prisma Client.
- `prisma:migrate`: يطبّق/ينشئ Migration باسم `init_catalog` أثناء التطوير.
- `prisma:seed`: يستعيد بصورة غير تدميرية كتالوج CSV الكامل وصوره وصفحات المحتوى والمقالات وسجل الإعدادات عند غيابها. لا يضيف طلبات أو عروض أسعار تجريبية، ولا يحذف السجلات أو يستبدل تعديلات الإدارة الموجودة.
- `prisma:studio`: يفتح أداة استعراض البيانات اختيارياً.

ملف CSV مخصص للاستيراد وSeed فقط، ولا يُقرأ كبديل وقت التشغيل. عند غياب `DATABASE_URL` أو تعذر الاتصال، يعرض المتجر حالة فارغة واضحة دون بيانات وهمية. توجد خريطة المصادر في `DATA_SOURCE_AUDIT.md`.

## لوحة إدارة الكتالوج

أضف القيم التالية إلى `.env`، مع كلمة مرور قوية وقيمة عشوائية مستقلة لـ`AUTH_SECRET`:

```env
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=replace-with-a-strong-password
AUTH_SECRET=replace-with-at-least-32-random-characters
```

المسارات:

- `/admin/login`: تسجيل الدخول.
- `/admin`: ملخص حالة الكتالوج.
- `/admin/products`: البحث وعرض المنتجات.
- `/admin/products/new`: إضافة منتج.
- `/admin/products/[id]/edit`: تعديل منتج.
- `/admin/categories`: إضافة وتعديل وحذف الأقسام وترتيبها.
- `/admin/brands`: إضافة وتعديل وحذف العلامات التجارية.
- `/admin/media`: مكتبة صور المنتجات المستخدمة والبحث فيها.
- `/admin/settings`: بيانات الشركة والرئيسية والهوية والتواصل والفوتر.
- `/admin/settings/payments`: اختيار مزود الدفع وحفظ مفاتيحه مشفّرة على الخادم.
- `/admin/customers`: قائمة المستخدمين وإدارة بياناتهم وأدوارهم للـSUPER_ADMIN.
- `/admin/blog`: إدارة المقالات التسويقية.
- `/admin/pages`: إدارة محتوى صفحات من نحن والعلامات والمشاريع وتجهيز المشاريع والستانلس والصيانة.
- `/admin/seo`: تدقيق النواقص والتكرار وإدارة SEO للصفحات العامة.
- `/admin/quotes`: قائمة عروض الأسعار وتغيير حالتها وتحويلها إلى طلب.
- `/admin/orders`: قائمة الطلبات وحالتها ورفع فاتورة PDF خارجية.

لا توفر لوحة الإدارة زر تعبئة بيانات تجريبية، ولا توجد API عامة لتشغيل Seed. حساب البيئة هو SUPER_ADMIN احتياطي لبدء التشغيل، وبعدها يمكن إنشاء ADMIN وSUPER_ADMIN داخل Neon. يبقى `AUTH_SECRET` في الخادم فقط، ويُشغّل الاستعادة يدوياً عند الحاجة.

### إدارة صفحات المحتوى

- الصفحات `/about` و`/brands` و`/projects` و`/project-solutions` و`/stainless` و`/maintenance` تقرأ محتواها المنشور من Prisma فقط.
- نموذج Prisma باسم `ContentPage` يحفظ العنوان والوصف والصورة والأزرار وأقسام الصفحة المرتبة.
- من `/admin/pages` يمكن تعديل الأقسام وإضافتها وحذفها وترتيبها واختيار الصور من مكتبة الوسائط، ثم نشر التغيير مباشرة.
- صفحة العلامات تعرض علامات المنتجات الحالية تلقائياً، وصفحة الستانلس تعرض منتجات قسم `stainless-steel`.
- لتفعيل الحفظ على قاعدة موجودة طبّق Migration الجديدة فقط؛ لا يلزم تشغيل Seed.

الجلسة موقعة وتنتهي بعد 8 ساعات، وتُحفظ في Cookie من نوع `httpOnly` و`sameSite=strict`. عمليات المنتجات والأقسام والعلامات تستخدم Prisma فقط. يمنع النظام حذف قسم أو علامة مرتبطة بمنتجات. عند غياب `DATABASE_URL` تعرض اللوحة حالة فارغة للقراءة فقط مع تعليمات التفعيل، دون عرض CSV كبيانات تشغيل.

بعد تحديث هذه النسخة على قاعدة موجودة، شغّل `npm run prisma:migrate` لإضافة حقلي بلد المنشأ والوصف الاختياريين للعلامة التجارية. لا يحتاج ذلك إلى إعادة Seed. تعيد عمليات الإدارة التحقق من صفحات الرئيسية والمتجر والبحث والأقسام والمنتجات وSitemap تلقائياً، كما تدعم صفحات Slug الجديدة العرض عند أول طلب دون Build جديد.

### المرحلة السابعة عشرة — منصة B2B وحسابات العملاء

- المسارات `/auth/login` و`/auth/register` و`/auth/verify` و`/auth/forgot-password` و`/auth/reset-password` و`/account` توفر حساب عميل بجلسة `httpOnly` موقعة، وتربط عروض الأسعار والطلبات الجديدة بالعميل تلقائياً.
- الحسابات الجديدة تتطلب رمز بريد من 6 أرقام صالحاً 10 دقائق، بينما تُعامل الحسابات الموجودة قبل Migration كحسابات موثقة حتى لا يتعطل أي عميل قديم. استعادة كلمة المرور تستخدم الرمز نفسه مع حد للمحاولات وإعادة الإرسال.
- لتفعيل الرسائل أضف `RESEND_API_KEY` و`EMAIL_FROM` في الخادم بعد توثيق الدومين لدى Resend. عند غياب الإعداد تتوقف التسجيلات الجديدة برسالة واضحة ولا تُنشأ جلسة غير موثقة.
- يعرض الحساب البيانات والطلبات والتتبع وإعادة الطلب والمفضلة، مع Pagination للطلبات.
- تحفظ `/admin/settings/payments` مفاتيح Mock أو Stripe أو Moyasar مشفّرة بـ AES-256-GCM؛ لا تُعاد المفاتيح إلى المتصفح ولا توجد معاملات دفع خارجية مفعلة.
- يسجل كل تغيير حالة طلب في Timeline؛ ملاحظات الإدارة داخلية ولا تظهر في `/track/[token]`.
- تعرض لوحة الإدارة إجمالي الطلبات والإيرادات وأفضل المنتجات والأقسام والطلبات اليومية، مع صفحات العملاء والمقالات.
- أضيفت مدونة Prisma قابلة للإدارة، واقتراحات بحث وتصحيح خفيف، وواجهة رئيسية مختصرة Search-first تعرض 4 أقسام وحتى 8 منتجات.
- أصبحت واجهة الكتالوج أكثر كثافة على الجوال: أقسام أفقية مدمجة وبطاقتان للمنتجات في الصف، مع الحفاظ على نسبة صور الأقسام `4:3` وصور المنتجات `1:1` دون تمديد أو قص.
- لتحديث قاعدة موجودة شغّل `npm run prisma:generate` ثم `npm run prisma:migrate` فقط. **لا تشغّل Seed**. تغيير `AUTH_SECRET` بعد حفظ مفاتيح الدفع يجعل القيم القديمة غير قابلة للفك.

### إدارة صور المنتجات

- من صفحة تعديل المنتج يمكن رفع صورة رئيسية أو حتى 8 صور إضافية في العملية الواحدة.
- الأنواع المدعومة: JPG وPNG وWebP، بحد أقصى 8 MB لكل صورة.
- تتحقق الخدمة من محتوى الملف، وتصحح الاتجاه، وتصغّر الأبعاد الكبيرة، ثم تحفظ WebP بجودة 86% داخل `public/images/products/uploads/`.
- يمكن تعديل Alt Text العربي، وتعيين الصورة الرئيسية، وتغيير الترتيب، والمعاينة والحذف بعد تأكيد واضح.
- الصور الأصلية الحالية لا تُحذف فعلياً؛ إزالة الصورة القديمة تحذف ارتباطها فقط. الملفات المرفوعة حديثاً تُحذف من التخزين عندما لا تبقى مرتبطة بأي منتج.
- يتطلب تفعيل الوسائط على قاعدة موجودة تشغيل `npm run prisma:migrate` لإنشاء `ProductImage` وتسجيل الصور الرئيسية الحالية تلقائياً. لا تشغّل Seed لهذا الغرض.

### مركز إعدادات الموقع

- يحتوي `/admin/settings` على تبويبات مبسطة لبيانات الشركة وHero والشعار وFavicon والتواصل والفوتر.
- تحفظ الإعدادات في سجل Prisma وحيد `SiteSettings`، وتنعكس على الهيدر والفوتر والرئيسية وصفحة التواصل مباشرة.
- صور Hero والشعار وFavicon تُختار من مكتبة الوسائط الحالية مع معاينة، بينما ألوان الهوية ثابتة.
- عند غياب PostgreSQL تُعرض قيم `siteConfig` للقراءة فقط، ويستمر الموقع دون تعطل.
- بعد تحديث نسخة موجودة شغّل `npm run prisma:migrate` فقط. لا تشغّل Seed لهذه المرحلة.

### SEO وأيقونات الأقسام

- تحتوي نماذج المنتجات والأقسام على تبويب SEO لعنوان الصفحة والوصف وCanonical وIndex/Noindex وOpen Graph وAlt Text.
- تعرض `/admin/seo` القيم التلقائية والتكرارات المحتملة، وتتيح تحرير SEO لكل صفحة عامة بشكل منفصل مع معاينة Google وعداد أحرف.
- تنشئ تغييرات Slug تحويلاً دائماً للعنوان القديم، وتُحدّث Metadata وSitemap فور الحفظ.
- تستخدم بطاقات الأقسام رسومات معدات SVG محلية واضحة، ويمكن اختيار رسم جاهز أو صورة من مكتبة الوسائط دون تغيير تصميم البطاقة.
- مساعد AI اختياري ويعرض اقتراحاً للمراجعة فقط ولا يحفظه. لتفعيله أضف:

```env
OPENAI_API_KEY=
OPENAI_MODEL=gpt-5-mini
```

يعمل SEO كاملاً عند غياب المفتاح. المفتاح لا يُحفظ في Prisma أو الكود. بعد تحديث قاعدة موجودة شغّل `npm run prisma:migrate` فقط، ولا تشغّل Seed.

### المرحلة التاسعة الخفيفة — عروض الأسعار والطلبات

- `/quote`: نموذج بسيط للاسم والهاتف والمدينة والملاحظات، مع اختيار المنتجات والكميات.
- `/admin/quotes`: عرض الطلبات وحالات «جديد / تم الرد / ملغي» والتحويل اليدوي إلى طلب.
- `/admin/orders`: حالات «جديد / قيد التنفيذ / تم التسليم / ملغي»، رفع PDF جاهز حتى 10 MB، وتسجيل دفعات داخلية خفيفة بدون بوابات خارجية.
- `/track/[token]`: متابعة آمنة برمز عشوائي، تعرض الحالة والمنتجات وملاحظات العميل وحالة الدفع فقط، وهي Noindex.
- تستخدم المرحلة نماذج Prisma: `Quote` و`QuoteItem` و`Order` و`OrderItem` و`Invoice` و`Payment`، ولا تولّد فواتير أو تربط Stripe/PayPal/Moyasar/STC Pay API.

لتفعيل الحفظ على قاعدة موجودة شغّل `npm run prisma:migrate` مرة واحدة. لا يحتاج ذلك إلى Seed. عند غياب قاعدة البيانات تعرض صفحات العروض والطلبات رسالة إعداد واضحة ولا تحفظ بيانات وهمية.

### المرحلة العاشرة — السلة وإكمال الطلب

- `/cart`: سلة محلية محفوظة في المتصفح، مع تعديل الكميات والحذف والإجمالي المبدئي.
- `/checkout`: اختيار طلب مباشر أو طلب عرض سعر وإدخال بيانات العميل دون حساب.
- الطلب المباشر متاح فقط عندما تكون أسعار جميع المنتجات محددة، ويُنشأ بحالة دفع «غير مدفوع» دون تحصيل إلكتروني.
- المنتجات بلا سعر ثابت تتحول لمسار عرض السعر ولا يمكن إنشاء إجمالي وهمي لها.
- يظهر مصدر الطلب داخل لوحة الإدارة: «طلب مباشر» أو «من عرض سعر».
- يستخدم الطلب المباشر نماذج المرحلة التاسعة نفسها داخل معاملة واحدة، مع سجل داخلي مخفي عن قائمة عروض الأسعار؛ لذلك لا يحتاج Stage 10 إلى Schema أو Migration إضافية.

يلزم فقط أن تكون Migrations المرحلة التاسعة مطبقة مسبقاً. لا تشغّل Seed لهذه المرحلة.

### المرحلة الحادية عشرة — نظام دفع Modular

- تمت إضافة طبقة دفع مستقلة عن المزود داخل `lib/payments`، ثم استُبدلت المزودات الثابتة بنظام المزودات الديناميكي الموضح في المرحلة الحادية والعشرين.
- تتوفر API Routes خفيفة: `POST /api/payments/create` و`POST /api/payments/verify`.
- تعرض لوحة الطلبات المزود، حالة الدفع، مرجع العملية، والمبلغ.

لتفعيل هذا التحديث على PostgreSQL شغّل Migration فقط. لا تشغّل Seed، ولا توجد بوابة دفع حقيقية في هذه المرحلة.

### المرحلة الثانية عشرة — Stripe / Moyasar Webhook Ready

- تم تحويل Prisma Client إلى الاستيراد القياسي من `@prisma/client`، وإزالة الاعتماد على `generated/prisma`.
- أصبح `Payment` مرتبطاً بطلب واحد، مع `webhookEventId` و`webhookReceivedAt` لتتبع استقبال الـ Webhook ومنع التكرار.
- تمت إضافة Webhook idempotent، وأصبح في المرحلة الحادية والعشرين يعمل مع اسم أي مزود مسجل بدلاً من قائمة ثابتة.
- تتم مزامنة الحالات مركزياً عبر `PaymentService`: `PENDING → PENDING_PAYMENT` و`PAID → CONFIRMED` و`FAILED/CANCELLED → CANCELLED`.
- تعرض قائمة الطلبات وتفاصيل الطلب مزود الدفع والحالة والمرجع والمبلغ وحالة Webhook.

لا يعتمد Checkout على اسم مزود. اختيار المزود وإدارته يتمان من `/admin/settings/payments`.

### المرحلة الثالثة عشرة — صفحات المحتوى

- أضيف `PaymentProvider` لإدارة أي مزود من `/admin/settings/payments`: الاسم، HTTPS Endpoint، المفاتيح المشفرة، والتفعيل أو الإيقاف. يسمح النظام بمزود نشط واحد.
- حُذفت تطبيقات Stripe/Moyasar/Mock الثابتة. يتواصل `PaymentService` فقط مع `HttpPaymentProvider` عبر عقد JSON موحد، ولا يعرف اسم أي شركة دفع.
- عند عدم وجود مزود نشط يُحفظ الطلب بصورة طبيعية بلا عملية دفع إلكتروني، ولا توجد مفاتيح أو اتصالات خارجية افتراضية.

لتطبيق التحديث شغّل `npm run prisma:generate` ثم `npx prisma migrate deploy`. لا تشغّل Seed.

عقد Endpoint الموحد يستقبل `POST` بالشكل `{ "action": "createPayment|verifyPayment|refundPayment|verifyWebhook", "data": { ... } }`. يجب أن يعيد إنشاء الدفع `transactionId` و`status`، وأن يعيد التحقق `status`. يرسل النظام المفتاح في `Authorization: Bearer` والسر الاختياري في `x-payment-secret`. تُقبل روابط HTTPS العامة فقط.

## كتالوج المرحلة الثانية

- `/shop`: بحث، فلتر قسم، فلتر توفر، ترتيب، وعدد نتائج.
- `/search`: بحث محلي بالاسم وSKU والموديل والعلامة والقسم.
- `/categories`: الأقسام الموجودة فعلياً في CSV (9 أقسام في الملف الحالي).
- `/categories/[slug]`: الأقسام الفرعية ومنتجات القسم.
- `/products/[slug]`: تفاصيل المنتج والمواصفات والملفات الفنية والمنتجات المشابهة.

يحفظ المتجر والبحث الحالة في Query Parameters، مثل:

```text
/shop?q=فرن&category=cooking-equipment&availability=in-stock&sort=price
```

زر واتساب يستخدم `siteConfig.whatsapp`. إذا بقي الرقم Placeholder فلن يُنشأ رابط خارجي، وسيظهر تنبيه محلي يشرح طريقة التفعيل.
